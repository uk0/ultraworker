## 2023-Q4 (10월~12월): 초기 POC 및 아이디에이션

### 주요 논의 사항
- **채널 목적**: Analyze 프로덕트를 위한 채널 (진형 개설)
- **초기 아이디에이션 미팅**: 진형, 시형, 석현 참석 (2023-09-27)
- **Analyze POC 개발**: 진형이 하이퍼클로바 기반으로 간단한 POC 제작
  - 엔드포인트: /sentiments, /summarize, /keywords, /categories -> /analyze로 통합
  - Golang으로 구현, Google Cloud Run에 배포
  - 스프레드시트 URL 입력 기반 분석
- **Jira 분리**: Analyze 전용 Jira 프로젝트 생성
- **현대 캐피탈 테스트**: raw 테스트 진행
- **Analyze 기획 초안**: 김혜원이 Figma에 기획서 작성
- **요약 기술 리서치**:
  - 43페이지 논문 요약 테스트: 의미 유사 단락별 split + 순서대로 입력 방식이 효과적
  - GPT-4 비용: 43페이지 기준 약 3,450원 (11회 호출)
  - CS 상담 스크립트 요약 테스트 진행
- **과금 모델 고민**: 한시형이 과금안 초안 작성
- **상품 패키징**: 한시형, 진형, 박우명 논의
- **네이버클라우드 연동**: BrightPattern(외산 콜센터 솔루션)에 분석 기능 연결 제안

### 주요 결정
- Analyze는 개별 기능마다 에이전트를 따로 띄우고 조합하는 형태로 개발
- 요약 프롬프트 최적화 방향 결정

## 2024-Q1 (1월~3월): CS Analyze 스킬 개발 및 테스트

### 주요 논의 사항
- **Analyze 요구사항 우선순위**: Google Sheets로 관리
- **테스트 데이터 제작**: 김혜원이 요약용 테스트 데이터셋 수작업 제작
  - 카테고리: 31개, 스크립트 데이터: 66건
- **임베딩 기반 카테고리 분류**: Qdrant에 임베딩 데이터 저장하여 유사도 기반 분류
- **CS Analyze 올인원 스킬 개발**:
  - 요약 + 키워드 추출 + 카테고리 분류 + 감정분석을 하나의 API로 통합
  - 하클(HyperCLOVA) 기반 요약으로 스타일 컨트롤
  - GPT 리랭커 역할로 카테고리 분류 (호출당 약 2원)
- **분석 품질 평가**:
  - 요약: "대체로 완벽"
  - 키워드: "꽤 잘 뽑음, 간혹 불필요한 것 포함"
  - 감정분석: "틀려보이는 건 없음"
- **CS 쉐어링 고객사 테스트**: STT 스크립트 기반 분석 테스트 진행
- **데모 사이트**: stargate-web.vercel.app/analyze에 데모 배포
- **Analyze 데모 제작**: 김덕현이 데모 제작 진행

### 주요 결정
- 요약에 예시 데이터를 few-shot으로 넣어 스타일 일관성 확보
- 키워드 추출은 요약된 내용 기반으로 수행하면 비용 절감 + 품질 향상
- 감정분석은 요약과 함께 묶어서 처리하여 호출 비용 절감
